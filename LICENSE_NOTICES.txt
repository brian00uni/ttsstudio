0. Public TTS API service (SaaS, no binary distribution)

Service:
https://tts.min-inter.co.kr

Operating model:
This public HTTP TTS API runs on the operator's own server (Docker container). Users receive
generated audio files and API responses only. FFmpeg binaries, Docker images, and server
application source code are not distributed to users or third parties.

FFmpeg usage:
FFmpeg is invoked server-side to convert synthesized WAV to MP3 (libmp3lame). FFmpeg is not
bundled in download ZIPs for end users in this deployment path. The operator does not publish
or convey FFmpeg binaries to API consumers.

License posture for this non-distribution model:
When FFmpeg is used only inside the operator's server and no FFmpeg binary, Docker image, or
install package containing FFmpeg is distributed to third parties, traditional GPL/LGPL
source-offer obligations tied to binary distribution generally do not arise for that usage.
This summary is practical guidance, not legal advice. If distribution plans change (Docker Hub,
on-premise delivery, client installers), review FFmpeg build flags and GPL/LGPL obligations again.

Runtime FFmpeg builds in this project:
- Docker service: Debian/Ubuntu package (apt install ffmpeg), typically LGPL-oriented distro build
- Local Windows dev/host (optional): Gyan.FFmpeg full build (GPLv3, --enable-gpl --enable-version3)

- Short credit: "This service uses FFmpeg (https://ffmpeg.org). FFmpeg is free software
  licensed under the LGPL/GPL. See /license for details."
