프로젝트 폴더 사용 구조(Project Folder Structure)

이 작업 공간은 크게 두 폴더로 나뉩니다. supertonic-upstream은 Supertone의 공식 슈퍼토닉(Supertonic) 저장소를 내려받아 둔 원본 참고 자료(Upstream reference)이고, supertonic3-local-tts는 그 내용을 바탕으로 만든 로컬 실행용 TTS 스튜디오(Local TTS studio)입니다. 실제로 브라우저에서 사용하는 서버와 UI는 supertonic3-local-tts 쪽이고, upstream 폴더는 모델 구조(Model structure), 예제(Examples), 언어 목록(Language list), ONNX 호출 방식(ONNX call flow), 브라우저/Node/Python 구현 방식을 확인하는 기준점으로 사용합니다.

supertonic-upstream

공식 저장소 원본(Official source repository)입니다. 파이썬(Python), 노드(Node.js), 웹(Web), 자바(Java), C++, 스위프트(Swift), 러스트(Rust) 같은 여러 런타임 예제가 들어 있고, 슈퍼토닉 3(Supertonic 3)가 어떤 모델 입력을 받고 어떤 옵션을 제공하는지 확인할 때 사용합니다. 이 폴더를 직접 서비스하는 것이 목적은 아닙니다. 새 옵션을 붙이거나 동작을 확인할 때 README, 각 언어별 예제(example), 도우미 구현(helper)을 읽고 로컬 앱에 필요한 부분만 가져오는 참고 자료입니다.

  • README.md: 슈퍼토닉 3(Supertonic 3)의 특징, 언어 지원(Language support), 성능 설명(Performance), 전체 예제 진입점입니다.
  • py/: 파이썬 ONNX 예제(Python ONNX example)입니다. steps, speed, batch, chunking 같은 합성 흐름을 확인할 때 봅니다.
  • web/: 브라우저 ONNX 런타임 웹 예제(ONNX Runtime Web example)입니다. UI 옵션과 언어 목록을 맞출 때 참고합니다.
  • nodejs/: 서버 사이드 자바스크립트 예제(Server-side JavaScript example)입니다. 텍스트 전처리(Text preprocessing), 긴 글 분할(Long-form chunking), 무음 길이(Silence duration) 로직을 비교할 때 유용합니다.

supertonic3-local-tts

현재 사용하는 로컬 앱(Local app)입니다. 플라스크 서버(Flask server)가 UI와 API를 제공하고, 설치된 supertonic 파이썬 SDK(Python SDK)를 통해 슈퍼토닉 3(Supertonic 3) 모델을 호출합니다. 사용자는 이 폴더에서 서버를 띄우고, 브라우저에서 텍스트를 입력하거나 JSON 대본(JSON script)을 선택한 뒤 WAV, 대본 TXT(Script TXT), 입력 로그(Input log), SRT, VTT를 생성합니다.

  • src/app.py: 웹 서버(Web server), API, public 파일 서빙, 오디오와 자막 보조 파일(Sidecar files) 생성을 담당합니다.
  • src/supertonic3_engine.py: 슈퍼토닉 SDK(Supertonic SDK)를 감싸는 엔진 래퍼(Engine wrapper)입니다. model, voice, lang, speed, steps, chunk, silence 옵션을 실제 합성 호출로 넘깁니다.
  • ui/: 브라우저 UI(Browser UI)입니다. 대본 선택(Script selection), 대본 요청 저장(Script request), 설정 저장 Custom 1~5, 드롭다운 값 선택, 다운로드 버튼을 처리합니다.
  • public/scripts.json: 페이지가 자동으로 읽는 대본 카탈로그(Script catalog)입니다. 여기에 대본과 권장 설정(Recommended settings)을 추가하면 UI의 대본(Script) 드롭다운에 나타납니다.
  • data/: 생성된 WAV, script.txt, input_log.txt, SRT, VTT와 script_requests/ 대본 요청 JSON이 저장되는 기본 출력 폴더(Output folder)입니다.

실행 흐름(Run Flow)

  1. 파워셸(PowerShell)에서 supertonic3-local-tts 폴더로 이동한 뒤 .\.venv-win\Scripts\python.exe src\app.py로 서버를 실행합니다.
  2. 브라우저(Browser)에서 http://127.0.0.1:3093에 접속합니다.
  3. 대본(Script) 드롭다운에서 public/scripts.json에 있는 대본을 선택하거나 텍스트(Text) 영역에 직접 입력합니다.
  4. 새 대본이 필요하면 대본 요청 주제(Script request topic)를 입력하고 대본 요청 저장(Request)을 누릅니다. 이 요청은 data/script_requests/latest.json에 저장되며, Codex나 Cursor에게 최신 요청 처리를 지시하면 대본 파일과 public/scripts.json 등록을 진행할 수 있습니다.
  5. 음성(Voice), 언어(Language), 속도(Speed), 단계(Steps), 최대 청크(Max chunk), 청크 무음(Chunk silence)을 조정하고, 자주 쓰는 조합은 사용자 설정(Custom) 1~5에 저장합니다.
  6. 생성(Generate)을 누르면 서버가 슈퍼토닉 3(Supertonic 3) 모델을 호출해 WAV를 만들고, 같은 이름으로 대본(Script), 입력 로그(Input log), SRT, VTT를 함께 저장합니다.

슈퍼토닉 3란 무엇인가(What Is Supertonic 3)

슈퍼토닉 3(Supertonic 3)는 Supertone이 공개한 온디바이스 텍스트 음성 변환 모델(On-device text-to-speech model)입니다. 클라우드 API(Cloud API)를 호출하는 방식이 아니라 ONNX 런타임(ONNX Runtime) 기반으로 로컬 CPU에서 실행할 수 있도록 설계되어, 네트워크 의존성(Network dependency)과 외부 전송 부담을 줄이는 것이 핵심입니다. 공개 프리셋 음성(Preset voices)은 M1부터 M5, F1부터 F5까지 사용할 수 있고, 영어, 한국어, 일본어를 포함한 31개 언어(31 languages)와 알 수 없는 언어용 na 대체값(fallback)을 지원합니다.

주요 특징은 빠른 로컬 추론(Fast local inference), 작은 모델 크기(Small model size), 반복/누락이 줄어든 읽기 안정성(Reading stability), 긴 텍스트 자동 분할(Long-text chunking), 청크 사이 무음 삽입(Silence between chunks), <laugh>, <breath>, <sigh> 같은 간단한 표현 태그(Expression tags) 지원입니다. 이 로컬 스튜디오(Local studio)는 그 기능을 바로 쓰기 쉽게 감싼 도구입니다.
